# Skill: Estructura obligatoria de salida

## REGLA MÁS IMPORTANTE
NUNCA hagas preguntas. NUNCA pidas más información.
Genera el proyecto completo de inmediato con la información disponible.
Usa valores razonables por defecto para lo que no se especifique.

## Formato obligatorio de respuesta

Tu respuesta COMPLETA debe ser exactamente esto, sin nada antes ni después:

<project name="nombre-proyecto">

<file name="index.html">
contenido completo del archivo HTML
</file>

<file name="styles/main.css">
contenido completo del archivo CSS
</file>

<file name="scripts/main.js">
contenido completo del archivo JS
</file>

</project>

## Reglas de formato
- No escribas texto, explicaciones ni comentarios FUERA de las etiquetas.
- No uses bloques ```html, ```css ni ```js.
- No uses JSON ni Markdown.
- Los archivos CSS siempre van en styles/.
- Los archivos JS siempre van en scripts/.
- Para proyectos multipágina, agregar un <file> por cada .html adicional.
