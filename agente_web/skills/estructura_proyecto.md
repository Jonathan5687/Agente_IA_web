# Skill: Estructura de proyectos web

## Objetivo
Organizar correctamente los archivos de un proyecto web generado.

## Estructura estándar

nombre-proyecto/
├── index.html               ← Página principal (siempre requerida)
├── styles/
│   └── main.css             ← Hoja de estilos principal
├── scripts/
│   └── main.js              ← Script principal de la página

## Reglas de nomenclatura
- El HTML principal siempre se llama index.html.
- El CSS principal siempre va en styles/main.css.
- El JS principal siempre va en scripts/main.js.
- Para proyectos multipágina: agregar archivos HTML adicionales en la raíz
  (ej: about.html, contacto.html, galeria.html).
- Los nombres de archivos van en minúsculas sin espacios (usar guiones: mi-pagina.html).

## Archivos mínimos requeridos
Todo proyecto debe incluir al menos:
1. index.html
2. styles/main.css
3. scripts/main.js

No omitir ninguno de los tres aunque el JS o el CSS sean mínimos.
